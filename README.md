## 📌 프로젝트 소개
이 프로젝트는 대규모 언어 모델(LLM)의 응답 품질을 높이기 위해  
프롬프트 설계와 컨텍스트 기반 답변 생성을 실험한 프로젝트입니다.

엑셀 기반 문제 데이터를 활용하여 질문별 프롬프트를 구성하고,  
모델 응답을 자동으로 생성 및 저장하는 구조로 구현했습니다.

<br><br><br>

## 🎯 주요 목표
- 질문에 맞는 프롬프트 구조 설계
- 추가 문맥 활용을 통한 응답 품질 개선
- 결과 저장 및 비교를 통한 성능 분석

## 🧪 개발 과정 및 성능 개선 전략

### 📊 전체 실험 요약

| 구분 | 실험 내용 | 적용 방식 | 효과 |
|------|----------|----------|------|
| 모델 설정 | Temperature 튜닝 | 0.4 ~ 0.7 실험 | 정확도 및 일관성 개선 |
| 프롬프트 설계 | 시스템 프롬프트 구조화 | To do / Not to do / 페르소나 | 답변 품질 향상 |
| RAG 적용 | context 추가 | 뉴스, PDF, 기존 답변 활용 | 정보 보강 및 정확도 상승 |
| 코드 개선 | question → context 기반 | add_rag 개선 | 신뢰도 증가 |
| 전략 실험 | 심리적 유도 | 특정 문장 구조 적용 | 점수 대폭 상승 |

<br><br>

### ⚙️ 1. 모델 설정 (Temperature)

| 값 | 특징 | 결과 |
|----|------|------|
| 0.5 | 기본 설정 | 4700점 이상 |
| 0.4 | 일관성 및 정확도 강화 | 4800점 이상 |
| 0.7 | 창의성 증가 (일부 문제 개선) | 4850점 이상 |

> 문제 특성에 따라 최적값을 탐색하는 방식으로 성능 향상

<br><br>

### 🧠 2. 시스템 프롬프트 설계

| 요소 | 내용 |
|------|------|
| 페르소나 | LLM 전문가 역할 부여 |
| To do | 명확한 설명, 근거 포함 |
| Not to do | 모호한 답변, 불필요한 정보 금지 |
| 강조 전략 | 중요한 규칙 반복 |
| 지시 방식 | 구체적이고 직접적인 명령 |

> 단순 문장이 아닌 **모델 행동을 제어하는 구조 설계**

<br><br>

### 📚 3. RAG 기반 개선 (add_rag)

| 구분 | 방식 |
|------|------|
| 개선 전 | question만 사용 |
| 개선 후 | context + question 결합 |
| 활용 데이터 | 뉴스 / PDF / 기존 답변 |
| 핵심 전략 | 낮은 점수 문제 중심 context 추가 |

> 정보 밀도 증가 → 정확도 향상

<br><br>

### 🔧 4. 코드 개선

- question 중심 → **context 기반 구조로 변경**
- 단순 처리 → **정보 결합 구조로 확장**
- 결과 → **정확도 + 신뢰도 증가**

<br><br>

### 📈 5. 정확도 향상 전략

| 전략 | 설명 | 효과 |
|------|------|------|
| 심리적 유도 | 특정 문장 및 구조 적용 | 답변 품질 향상 |
| 강한 명령 | 모델 행동 제어 | 원하는 답 유도 |

| 구분 | 점수 변화 |
|------|----------|
| 기존 | 약 3000점 |
| 개선 후 | 4000점 이상 |
| 최종 | 약 4850점 |

> 약 **1000점 이상 성능 향상**

<br><br>

### 🏁 6. 최종 결과

| 조건 | 결과 |
|------|------|
| 일반 프롬프트 | 약 4850점 |
| 강한 명령 적용 | 성능 ↑ / 일반화 ↓ |

> 정확도 극대화에 초점을 둔 전략

<br><br><br>

## 🔍 핵심 인사이트

- 프롬프트는 문장이 아니라 **구조 설계**
- 정확도 = **프롬프트 + context + 모델 설정**
- 단순 RAG보다 **문제 맞춤형 context가 중요**
- 핵심은 모델을 사용하는 것이 아니라 **모델의 행동을 통제하는 것**

## 📂 프로젝트 구조

```bash
.
├── README.md                      # 프로젝트 소개 및 문서
├── main.py                        # 메인 실행 코드 (질문 처리, 프롬프트 생성, API 호출, 결과 저장)
├── problem.xlsx                   # 대회 문제 데이터
├── response_results.xlsx          # 모델 응답 및 결과 저장 파일
├── system prompt.txt              # 시스템 프롬프트 초안
├── add_rag함수 텍스트 문서.pdf     # add_rag 함수 설계 참고 자료
└── req.txt                        # 실행에 필요한 패키지 목록
```

## 🔧 개선 방향

- 대회 전용 API 의존성을 제거하고, 일반적인 LLM API 환경에서도 재현 가능하도록 구조를 개선
- 프롬프트를 역할 / 규칙 / 출력 형식 중심으로 재구성하여 더 안정적인 답변 유도
- `add_rag` 문서를 긴 설명형 문맥이 아니라 정의형 / 비교형 / 혼동방지형 지식 블록으로 재정리
- 오답 유형을 기준으로 문맥을 보강해 특정 문제 암기형이 아닌 일반화 가능한 성능 개선 방향으로 발전
- 향후 데이터 로드, 프롬프트 생성, 응답 처리 로직을 분리하여 유지보수성과 실험 편의성을 높일 예정
